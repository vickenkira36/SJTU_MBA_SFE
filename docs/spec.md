# 分配前后比对功能 Spec

## 问题陈述

当前 ResultsView 中的 HistoricalComparison 组件仅提供医院归属变动的简单统计（新增/移除/保持），缺少辖区级指标对比、均衡性统计、可视化图表和 Excel 导出。业务部门需要一个完整的前后比对方案，覆盖辖区级和医院级两个维度，同时支持页面可视化和 Excel 导出。

## 数据来源

- **历史数据**：用户上传的历史分配 Excel（`HistoricalAssignment[]`），包含 inscode → trtyCode 的映射和可选的比例字段
- **当前数据**：算法优化后的 `OptimizationResult`，包含 assignments 和 territoryResults
- **医院主数据**：`Hospital[]`，提供 index、销量、潜力、城市、省份等字段
- **辖区主数据**：`Territory[]`，提供 trtyCode、rep、province、region、lel 等字段

## 需求

### 1. 辖区级对比

为每个辖区计算历史和当前两组指标并对比：

| 指标 | 历史值来源 | 当前值来源 |
|------|----------|----------|
| Index 总值 | 历史分配的医院 index 之和 | territoryResult.totalIndex |
| 医院数量 | 历史分配的 inscode 去重数 | territoryResult.hospitalCount |
| 城市数量 | 历史医院的城市去重数 | territoryResult.cityCount |
| 销量总值 | 历史医院的销量之和 | territoryResult.totalSales |
| 潜力总值 | 历史医院的潜力之和 | territoryResult.totalPotential |

每个指标显示：历史值、当前值、差异值（当前-历史）、变化率。

#### 辖区变化度量

定义：一个辖区中所有发生变动的医院（移除 + 新增 + 变更）的 index 总和。

示例：辖区 A 移除了 3 家医院（index 合计 800），新增了 2 家医院（index 合计 600），则辖区变化度量 = 800 + 600 = 1400。

- 变化度量越大，说明该辖区的人员调整幅度越大
- 辖区级对比表默认按变化度量从高到低排序
- 变化度量为 0 的辖区（无任何变动）排在最后

#### 下钻交互

辖区级对比表中每行可点击展开，查看该辖区的变动明细：
- 保持的医院列表（inscode、insname、index）
- 新增的医院列表（inscode、insname、index、来源：新分配）
- 移除的医院列表（inscode、insname、index）
- 变更的医院列表（inscode、insname、index、历史辖区 → 当前辖区）

下钻明细内按 index 降序排列。

### 2. 医院级对比

逐家医院列出归属变化：

| 字段 | 说明 |
|------|------|
| inscode | 医院代码 |
| insname | 医院名称 |
| 省份 | 医院所在省份 |
| 城市 | 医院所在城市 |
| index | 医院 index 值 |
| 历史辖区 | 历史分配的 trtyCode（可能多个） |
| 当前辖区 | 当前分配的 trtyCode（可能多个） |
| 变动类型 | 保持 / 新增 / 移除 / 变更 |

变动类型定义：
- **保持**：历史和当前归属同一辖区
- **新增**：历史无分配，当前有分配
- **移除**：历史有分配，当前无分配
- **变更**：历史和当前归属不同辖区

排序规则：默认按 index 降序排列（大 index 医院排在前面）。页面和 Excel 导出均使用此排序。

### 3. 均衡性统计指标

对比历史和当前的整体均衡性：

| 统计量 | 说明 |
|--------|------|
| Index 平均值 | 所有辖区 index 的平均 |
| Index 标准差 | 衡量离散程度 |
| Index 极差 | 最大值 - 最小值 |
| Index CV | 变异系数 = 标准差 / 平均值 |
| 达标率 | index 在目标范围内的辖区占比 |

### 4. 页面可视化

替换现有的 HistoricalComparison 组件，新组件包含：

#### 4.1 汇总卡片
- 变动率（变更+新增+移除 占总医院数的比例）
- 均衡性改善（CV 变化：历史 CV → 当前 CV）
- 达标率变化（历史达标率 → 当前达标率）

#### 4.2 辖区 Index 对比图
- 分组柱状图：每个辖区两根柱子（历史 index vs 当前 index），按当前 index 降序排列
- 叠加目标范围参考线（indexMin / indexMax）
- 辖区数量多时支持横向滚动

#### 4.3 辖区对比表（含下钻）
- 每行显示：辖区代码、Rep、历史 index、当前 index、变化度量、保持/新增/移除/变更数量
- 默认按变化度量降序排列
- 点击行展开下钻明细：保持/新增/移除/变更的医院列表，按 index 降序
- 默认显示前 20 个辖区，可展开全部

#### 4.4 均衡性对比表
- 两列对比表格：历史 vs 当前，每行一个统计量
- 改善的指标标绿，恶化的标红

### 5. Excel 导出

导出一个多 sheet 的 Excel 文件，包含：

| Sheet 名 | 内容 |
|----------|------|
| 辖区对比 | 辖区级对比表（每行一个辖区，含变化度量、各指标的历史值/当前值/差异，按变化度量降序） |
| 医院变动明细 | 医院级对比表（每行一家医院，含变动类型和前后辖区） |
| 均衡性统计 | 均衡性指标对比（历史 vs 当前） |

## 验收标准

1. 上传历史分配数据后，结果页显示完整的前后比对模块（替换原 HistoricalComparison）
2. 辖区级对比图表正确显示每个辖区的历史和当前 index
3. 医院级变动明细正确分类为保持/新增/移除/变更
4. 均衡性统计指标（CV、标准差、达标率）计算正确
5. Excel 导出包含三个 sheet，数据与页面展示一致
6. 无历史数据时，对比模块不显示
7. 省份筛选生效时，对比仅基于选中省份的数据

## 实施步骤

1. **构建历史辖区指标计算函数**：从 `HistoricalAssignment[]` + `Hospital[]` 反推每个历史辖区的 index/医院数/城市数/销量/潜力
2. **构建医院变动分析函数**：逐家医院对比历史和当前归属，分类为保持/新增/移除/变更
3. **构建均衡性统计函数**：计算历史和当前的 CV、标准差、极差、达标率
4. **实现辖区 Index 对比图**：Recharts 分组柱状图，历史 vs 当前
5. **实现汇总卡片**：变动率、CV 变化、达标率变化
6. **实现变动明细表**：按辖区分组的医院变动列表，支持折叠/展开
7. **实现均衡性对比表**：两列对比表格
8. **实现 Excel 导出**：三个 sheet 的对比报告
9. **删除旧 HistoricalComparison 组件**
10. **验证**：用测试数据验证所有指标计算和展示
