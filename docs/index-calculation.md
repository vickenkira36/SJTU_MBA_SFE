# Index 计算说明

## 公式

```
index = (销量归一化 × 销量权重 + 潜力归一化 × 潜力权重) × 省份人头数 × 1000
```

## 字段定义

| 字段 | 含义 | 计算方式 |
|------|------|---------|
| 销量归一化 | 该医院销量占所在省份的百分比 | 医院销量 / 省份所有医院销量之和 |
| 潜力归一化 | 该医院潜力占所在省份的百分比 | 医院潜力 / 省份所有医院潜力之和 |
| 销量权重 | 销量在综合指标中的预设占比 | 用户设定，与潜力权重之和 = 100% |
| 潜力权重 | 潜力在综合指标中的预设占比 | 用户设定，与销量权重之和 = 100% |
| 省份人头数 | 该省份的辖区（territory）数量 | 统计该省份的 TRTY_CODE 数 |

## 常数

`1000` 表示理想状态下每个辖区的 index 总值。

## 性质

- 同一省份所有医院的 index 之和 = 省份人头数 × 1000
- 若医院均匀分配到各辖区，每个辖区的 index 总值恰好为 1000
- index 同时反映销量贡献和潜力贡献，权重比例由用户决定

## 示例

假设湖南省有 20 个辖区，销量权重 60%，潜力权重 40%：

某医院销量归一化 = 0.03，潜力归一化 = 0.02

```
index = (0.03 × 0.6 + 0.02 × 0.4) × 20 × 1000
     = (0.018 + 0.008) × 20000
     = 0.026 × 20000
     = 520
```

该省所有医院 index 之和 = 20 × 1000 = 20000。

## 为什么这么计算 Index

### 为什么用销量 + 潜力，而不是单一指标？

销量反映的是医院当前的业务贡献，潜力反映的是未来的增长空间。如果只看销量，高潜力但目前销量低的医院会被低估，代表可能错过增长机会；如果只看潜力，已经贡献大量销量的医院会被忽视，代表的工作量无法体现。

将两者加权组合，可以让每个辖区既有稳定的业务基础（销量），又有合理的成长空间（潜力）。权重比例由业务部门根据当期策略灵活设定——偏重维护现有业务时提高销量权重，偏重市场开拓时提高潜力权重。

### 为什么要做归一化（除以省份总量）？

不同省份的市场规模差异很大。比如上海的医院销量可能是青海的几十倍。如果直接用原始销量数字，跨省份的医院之间无法公平比较。

归一化后，每家医院的数值变成"占所在省份的百分比"。这样无论省份大小，每家医院的 index 都反映它在本省内的相对重要性，而不是绝对金额。

### 为什么要乘以省份人头数 × 1000？

归一化之后，同一省份所有医院的百分比加起来等于 100%。但我们需要一个有实际意义的数字来衡量辖区的工作量是否均衡。

乘以"省份人头数 × 1000"的作用是：

- **省份人头数**（即该省的辖区数量）让 index 总量与辖区数量挂钩。如果一个省有 20 个辖区，那么该省所有医院的 index 之和就是 20 × 1000 = 20000。
- **1000** 是一个基准值，代表"理想状态下每个辖区应该承担的 index"。如果分配完全均匀，每个辖区的 index 总值恰好是 1000。

这样设计的好处是：拿到任何一个辖区的 index 总值，就能直观判断它的负荷——接近 1000 说明均衡，远高于 1000 说明负担过重，远低于 1000 说明负担过轻。不需要额外换算或对比其他数字。
